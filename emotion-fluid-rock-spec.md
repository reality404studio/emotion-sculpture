# Emotion Sculpture — Fluid Rock Visual Specification

> 감정 데이터가 조각의 선이나 레이어가 아니라, 조각을 이루는 재료 자체가 된다.

## 0. 목적

현재 결과물은 감정 데이터가 세로 링과 색 리본으로 외부에 드러나는 시각화에 가깝다. 이 명세의 목표는 이를 버리고, 2번·3번 레퍼런스처럼 하나의 유리·레진 덩어리 안에 색, 흐름, 굴절, 압력이 갇힌 결과물을 만드는 것이다.

최종 결과는 사용자가 입력을 무작위로 해도 항상 같은 미적 범위 안에 있어야 한다.

핵심 문장:

> 감정이 조각 위에 그려지는 것이 아니라, 감정이 조각의 재료가 된다.

---

## 1. DON'T — 반드시 피해야 하는 것

아래 항목은 현재 1번 결과의 시각 문법이며, 다음 구현에서 금지한다.

### 형태

- 세로로 긴 타워·토템·화병 형태
- 시간 틱마다 수평 링을 위로 쌓는 구조
- 일정한 원통, 컵, 꽃병처럼 읽히는 실루엣
- 위아래가 분리된 층, 캡, 계단, 띠
- 모든 입력이 전체 반지름을 키우거나 줄이는 방식
- 감정에 따라 높이가 계속 자라는 구조

### 감정 표현

- 금·빨강·파랑 선을 오브제 바깥에 직접 감는 방식
- 감정별 Tube/Line이 데이터 시각화 선처럼 주인공이 되는 방식
- 세 감정이 모두 같은 반지름 변형의 다른 버전으로 보이는 방식
- 색깔만 다르고 동작은 같은 세 버튼
- 색을 RGB 평균내서 보라색·회색·흰색으로 뭉개는 방식

### 표면과 재질

- 저해상도 삼각형, 노이즈, 사출 자국이 표면에 드러나는 것
- 작은 지터를 많이 넣어 질감이 복잡해 보이게 하는 것
- 하얀 플라스틱이나 석고처럼 전체가 뽀얗게 뜨는 것
- 색이 표면 위에 스티커처럼 올라오는 것
- 얇은 선과 점만으로 내부 깊이를 표현하는 것

### 인터랙션

- 버튼 입력 후 다음 0.7초 틱까지 시각적 반응을 기다리는 것
- 버튼을 누를 때마다 전체 지오메트리를 무겁게 재생성하는 것
- 카메라가 오브제의 성장에 맞춰 계속 쫓아가며 프레이밍을 흔드는 것
- 무작위 입력에 따라 실루엣이 망가지는 것

## 2. TARGET — 2번·3번에서 가져올 공통 문법

### 2.1 하나의 닫힌 유체 덩어리

오브제는 여러 파츠의 집합이 아니라, 하나의 유리·레진 덩어리로 읽혀야 한다.

- 화면상 가로·세로 비율은 약 0.85~1.15
- 화면 높이의 58~72%를 차지하는 단일 히어로 오브제
- 낮고 넓은 비정형 바위 형태
- 완전한 구형은 피하되, 3~5개의 큰 굴곡으로 구성
- 비대칭이지만 전체 중심은 안정적
- 위·아래가 잘린 원통이 아니라 둥글게 이어지는 외곽

### 2.2 내부에 갇힌 흐름

움직임은 외부 선이 아니라 덩어리 내부의 색과 굴절로 보여야 한다.

- 큰 색 영역이 먼저 보임
- 가까이에서 내부 소용돌이와 마블링이 보임
- 색은 표면을 칠하는 것이 아니라 깊이를 가진 층으로 존재
- 하나의 색맥이 오브제의 앞·옆·뒤를 관통하는 것처럼 보여야 함
- 회전하면 내부 색의 깊이와 반사 위치가 달라져야 함

### 2.3 큰 흐름 + 작은 디테일

시각 정보의 우선순위는 다음과 같다.

```text
큰 실루엣
  → 큰 색 덩어리
    → 내부 소용돌이
      → 유리 하이라이트와 작은 입자
```

작은 노이즈나 파티클이 실루엣보다 먼저 보이면 실패로 간주한다.

### 2.4 광학적 재질

본체는 하얀 플라스틱이 아니라 색이 안쪽에 머무는 유리·레진이어야 한다.

목표 재질 특성:

- 높은 반사율
- 낮은~중간 거칠기
- 부드러운 Fresnel 가장자리
- 얇은 외곽과 두꺼운 중심의 밝기 차이
- 내부 컬러 레이어의 깊이감
- 표면에 길게 흐르는 하이라이트
- 바닥에 부드러운 접지 그림자

권장 시작 범위:

```text
roughness: 0.08 ~ 0.22
clearcoat: 0.55 ~ 0.85
transmission: 0.18 ~ 0.55
ior: 1.35 ~ 1.48
```

수치는 고정 목표가 아니라 레퍼런스와 화면 밝기에 맞춘 튜닝 범위다.

## 3. 감정 데이터 모델

감정은 본체 실루엣 전체를 바꾸는 값이 아니라, 내부 재료 필드의 파라미터가 된다.

```text
baseShape = deterministicFluidRock(seed)
materialField = f(seed, beats, emotion)
liveImpulse = f(currentInput, emotion)
```

### 3.1 본체 실루엣

- 세션 시작부터 완성된 Fluid Rock 형태가 존재한다.
- 모든 세션은 같은 아름다운 형태 범위 안에서 시작한다.
- 세션 seed는 비대칭, 굴곡 방향, 큰 볼륨 분포만 결정한다.
- 감정 데이터는 본체 실루엣을 최대 ±8~12% 이내에서만 보정한다.
- 입력을 아무렇게 해도 구형·원통·뾰족한 가시 형태로 무너지지 않아야 한다.

### 3.2 내부 재료 필드

감정별로 하나의 큰 컬러 필드와 흐름 방향을 가진다.

```text
joyField     = warm bloom / gold-orange flow
noField      = red-pink rupture / compressed fold
pleaseField  = blue-cyan pressure / slow membrane
```

각 필드는 다음 속성을 가진다.

- 위치
- 폭
- 깊이
- 흐름 방향
- 투명도
- 발광량
- 굴절 왜곡
- 표면 거칠기

색은 세 필드를 먼저 각각 계산한 뒤, 재질 내부에서 겹친다. 최종 표면 색을 단순 평균해서 만들지 않는다.

## 4. 감정별 시각 문법

세 감정은 반드시 서로 다른 물리 현상처럼 보여야 한다.

### 4.1 좋아! — Bloom

입력: 빠른 연타

시각 동사: 피어난다 / 번진다 / 솟아난다

- 골드·앰버·오렌지 컬러 필드
- 내부의 따뜻한 색층이 바깥으로 퍼짐
- 여러 번 누를수록 작은 꽃잎형 색 덩어리와 하이라이트 증가
- 표면은 부드럽게 열리고, 날카로운 균열은 만들지 않음
- 반응은 빠르고 밝으며, 잔상은 짧음

입력 피드백:

```text
attack: 즉시
shape: outward bloom
color: warm gold/orange
decay: 300~500ms
```

### 4.2 안돼! — Rupture

입력: 단발 탭

시각 동사: 찢어진다 / 꺾인다 / 눌린다

- 레드·코랄·핑크 컬러 필드
- 한 번의 탭마다 하나의 강한 접힘 또는 압력 자국
- 내부 색층이 짧은 시간 동안 갈라져 보임
- 얇은 선을 그리는 것이 아니라 넓은 어두운 면과 밝은 가장자리를 만듦
- 한 번 발생한 뒤 빠르게 안정되어야 함

입력 피드백:

```text
attack: 즉시
shape: inward fold / sharp compression
color: red/pink
decay: 450~700ms
```

### 4.3 제발! — Breathe

입력: 홀드

시각 동사: 호흡한다 / 감싼다 / 압력을 유지한다

- 블루·시안·인디고 컬러 필드
- 투명한 외곽막 또는 깊은 내부층이 천천히 팽창·수축
- 누르고 있는 시간에 비례해 필드의 폭과 깊이가 증가
- 손을 떼면 즉시 사라지지 않고 길게 안정됨
- 다른 감정처럼 번쩍이거나 찢어지지 않음

입력 피드백:

```text
attack: 100~180ms
shape: slow breathing pressure
color: blue/cyan/indigo
release: 900~1400ms
```

## 5. 시간 기록 방식

시간은 세로 높이로 표현하지 않는다.

### 5.1 기록 데이터

- beats는 기존처럼 결정론적으로 저장한다.
- beat 데이터는 고정된 3D 내부 필드의 방향·위치·세기·밀도를 조정한다.
- 긴 경기는 32~64개의 내부 control slice로 리샘플링한다.
- 모든 beat를 수평 링으로 직접 렌더하지 않는다.

### 5.2 결과물의 변화

- 경기 초반: 얕은 색층과 작은 내부 흐름
- 긴장 구간: 블루 필드의 깊이와 압력 증가
- 안돼 구간: 레드 접힘과 어두운 내부 공간 생성
- 좋아 구간: 골드 필드의 확산과 하이라이트 증가
- 최종 결과: 하나의 안정된 Fluid Rock 안에 모든 감정의 흔적이 남음

### 5.3 결정론

동일한 `seed + beats`는 동일한 결과를 만들어야 한다.

- base silhouette
- 내부 컬러 필드
- 소용돌이 방향
- 밝은 하이라이트 위치
- 최종 해시용 데이터

모두 결정론적이어야 한다. `Math.random()`은 사용하지 않는다.

## 6. 즉시 입력 반응

입력 반응과 기록 반응을 분리한다.

```text
button input
  ├─ liveImpulse: 즉시 화면 반응
  └─ beat snapshot: 다음 tick에서 영구 기록
```

### liveImpulse

- 버튼 입력 후 한 프레임 안에 시작
- 본체 지오메트리 전체를 재생성하지 않음
- shader uniform 또는 별도의 내부 재질 레이어로 처리
- 색, 투명도, 굴절, 하이라이트, 압력만 즉시 변화

### beat snapshot

- 기존 0.7초 주기로 저장
- 결과물 재현과 해시에 사용
- 라이브 임펄스의 애니메이션 시간값은 해시에 포함하지 않음

---

## 7. 카메라와 조명

### 카메라

- 오브제를 화면 중앙에 고정
- 경기 중 카메라가 높이를 따라 올라가지 않음
- 느린 자동 회전만 허용
- 결과 화면에서 오브제 전체가 한 번에 보임
- 오브제가 화면 높이의 58~72%를 차지함

### 조명

- 밝은 회색 또는 흰색 배경
- 부드러운 큰 키 라이트
- 차가운 블루 필 라이트
- 따뜻한 골드 림 라이트
- 바닥의 부드러운 그림자와 반사
- 포스트프로세싱이 배경을 검게 만드는 방식은 사용하지 않음

---

## 8. 아름다움 보장 규칙

임의의 입력에서도 아래 규칙을 만족해야 한다.

1. 입력이 없어도 Fluid Rock처럼 보인다.
2. 입력이 많아져도 본체 실루엣은 같은 미적 계열에 머문다.
3. 1초 안에 유리·레진·액체 바위로 인식된다.
4. 세 감정의 반응이 색뿐 아니라 움직임으로 구분된다.
5. 외부에 데이터 선이 감기지 않는다.
6. 링·층·원통·화병으로 읽히지 않는다.
7. 최소 3개의 큰 색 영역과 1개의 깊은 어두운 영역이 존재한다.
8. 색은 넓은 면과 내부 깊이로 보이고, 얇은 선에 의존하지 않는다.
9. 랜덤성은 큰 흐름의 방향에만 적용하며, 고주파 노이즈는 제한한다.
10. 회전하면 앞면과 뒷면의 내부 재료가 다르게 보인다.

---

## 9. 데모 연출 기준

### 시작

입력이 없어도 조용한 Fluid Rock이 천천히 호흡한다.

### 제발!

파란 내부막이 부드럽게 팽창하면서 오브제를 감싼다.

### 안돼!

레드·핑크 압력이 한 번 깊게 접히며 내부에 어두운 굴곡을 만든다.

### 좋아!

골드·오렌지 흐름이 빠르게 퍼지고 표면 하이라이트가 꽃처럼 열린다.

### 완성

모든 반응이 안정된 뒤, 하나의 유리·레진 오브제 안에 세 감정의 색과 흐름이 남는다.

---

## 10. 구현 순서

1. 감정 데이터를 연결하지 않은 Fluid Rock 본체 제작
2. 본체의 실루엣과 재질만으로 레퍼런스 수준의 정지 화면 확보
3. 세 개의 내부 컬러 필드 제작
4. 좋아·안돼·제발의 서로 다른 동작 구현
5. 즉시 liveImpulse 연결
6. beat 데이터를 내부 필드에 리샘플링
7. 결정론적 재현과 해시 검증
8. 기존 비교 뷰와 민팅 화면 연결

본체가 입력 없이도 아름답지 않으면 다음 단계로 진행하지 않는다.
