# Emotion Trophy — 비주얼 문법 명세 (v3.1)

> **v3.1 파티클 개정** — §10 참조. 아키타입·시간축·감정 문법은 유지하되,
> 재료가 유리 메시에서 입자로 바뀌고 무대가 스타디움 나이트로 전환됐다.

> 골은 스코어보드에 남는다. 열정에는 트로피가 없었다.
> 이제 경기를 보는 90분 동안, 모든 팬의 감정이 자기만의 트로피를 주조(casting)한다.

## 0. 목적함수

이 명세의 모든 규칙은 아래 세 문장에서 파생된다. 충돌 시 이 순서가 우선순위다.

1. **무엇을 입력하든 아름답다.** 스팸, 무입력, 한쪽 감정만 연타 — 어떤 입력 분포에서도 결과물은 전시 가능한 트로피 범위 안에 있어야 한다.
2. **시간의 흐름이 보인다.** 시간은 세로축(아래 = 킥오프, 위 = 종료 휘슬)으로 표현되고, 흔적이 남은 구간과 남지 않은 구간이 한눈에 구분되어야 한다. 고요도 기록이다.
3. **내 입력이 확실하게 남는다.** 감정별로 색·부피·무늬·모션이 모두 다르게, 입력 즉시, 직관적으로 보여야 한다.

## 0.1 이전 버전에서 계승·폐기하는 것

| | 링 토템 (v1) | Fluid Rock (v2) | Trophy (v3) |
|---|---|---|---|
| 목적 1 아름다움 | ✗ 데이터 시각화로 보임 | ○ 재질은 성공 | v2의 유리·레진 광학 재질 계승 |
| 목적 2 시간 가독 | ○ 세로 = 시간 | ✗ 덩어리 속에 묻힘 | v1의 세로 시간축 + 세로 링 로프팅 파이프라인 계승 |
| 목적 3 인과 가독 | △ 남긴 하나 조잡함 | ✗ 어디에 남았는지 안 보임 | 주조선(성장면) 개념으로 재설계 |
| 정체성 | 정체불명 토템 | 정체불명 돌 | **트로피** — 월드컵 컨셉과 직결 |

v1이 못생겼던 원인은 시간축이 아니라 실행이었다: 저해상도 스파이크, 원색 리본, 층이 분리된 원통 실루엣. v3는 시간축을 유지하되 **트로피 프로파일이라는 강한 형태 규범**으로 아름다움의 하한선을 만든다.

## 1. 핵심 개념 — 트로피 주조 (Trophy Casting)

경기가 시작되면 빈 받침대 위로 녹은 금빛 유리가 차오르며 트로피가 아래에서 위로 주조된다.

- **주조선(성장면) = 현재.** 차오르는 최상단에 얇게 달아오른 용융 링이 있고, 지금 이 순간의 감정은 여기에만 개입한다.
- **주조선 아래 = 과거.** 굳은 재료는 어떤 입력으로도 소급해서 바뀌지 않는다.
- **입력 → 주조선이 즉시 출렁임 → 몇 초 뒤 그 형태로 응고 → 트로피 표면과 내부에 영구히 남음.**
- **종료 휘슬 = 주조 완료.** 림까지 차오르면 트로피가 완성되고, 카메라가 빠지며 전체가 공개된다.

이 하나의 이미지가 세 목적함수를 동시에 해결한다: 인과가 주조선 한 곳에 모이고(목적 3), 굳은 층이 시간의 기록이 되고(목적 2), 트로피 프로파일 + 응고 평활화가 아름다움을 보장한다(목적 1). 그리고 "내 열정으로 주조한 나만의 우승컵"이라는 서사가 민팅(영구 기록)과 정확히 맞물린다.

## 2. 형태 문법 — 트로피 해부

### 2.1 프로파일 (모든 트로피의 공통 뼈대)

트로피는 고정된 아키타입 프로파일 곡선 `P(y)` 를 가진다. 감정은 이 곡선을 **±15~25% 범위 안에서만** 변조한다. 프로파일이 아름다움의 하한선이다.

```text
      ┌ 림(rim)        높이  92~100%   살짝 벌어지는 입구
      │ 컵 몸통(bowl)   높이  40~92%   아래가 좁고 위가 넓은 포물선형 컵
      │ 스템(stem)      높이  18~40%   가는 기둥, 중앙에 살짝 볼록한 노트(knot)
      │ 받침대(base)    높이   0~18%   넓고 낮은 2단 원반 — 시간축에서 제외
```

- 손잡이는 만들지 않는다. 스템 + 컵 실루엣만으로 트로피로 읽히며, 손잡이는 지오메트리 파탄 위험만 키운다.
- 전체 비례: 폭 : 높이 ≈ 1 : 1.9~2.3. 화면 높이의 70~80%를 차지하는 단일 히어로 오브제.
- **받침대는 감정과 무관한 고정 지오메트리.** 완성 화면에서 매치 참조와 해시 축약이 각인되는 자리다(민팅 연결).
- 위·아래 경계(받침대↔스템, 컵↔림)는 단차 없이 곡률로 이어진다. 층·캡·띠로 읽히면 실패.

### 2.2 시간 매핑

- **킥오프 = 스템 시작(높이 18%), 종료 휘슬 = 림(높이 100%).** 경기 길이와 무관하게 이 구간에 균등 매핑 — 틱당 높이가 일정하므로 시간이 선형으로 읽힌다.
- 하프타임 지점(높이 ≈ 59%)에는 아주 미세한 결(내부 색온도 변화 한 번)만 남긴다. 지오메트리 단차 금지.
- 스팸/무입력 어느 쪽이든 최종 높이는 동일하다. 차이는 두께·색·무늬로만 남는다.

### 2.3 감정 변조와 응고 평활화

- 반지름: `R(y,θ) = P(y) × (1 + Σ emotionLobes(y,θ))`, 결과는 `[0.78, 1.25] × P(y)` 로 클램프.
- 주조선에서는 거칠게 출렁여도 되고, **굳을 때 다듬어진다**: 이벤트 하나는 응고되며 세로로 3~6틱 두께로 번진다(가우시안, 위쪽 비대칭 — 위로 새어나온 흔적이 "번짐"으로 읽힘).
- 둘레 방향 최소 피처 폭 ≈ 24°. 바늘 스파이크 금지 — 뾰족함이 필요한 감정도 "주름"으로 표현하지 "가시"로 표현하지 않는다.
- 고주파 지터로 질감을 만들지 않는다. 질감은 재질(§5)이 담당한다.

### 2.4 고요의 형태

- 입력 없는 구간: `P(y)` 그대로의 매끈하고 맑은 유리. 결정론적 미세 맥동(±2%)만.
- 고요는 숨기지 않는다 — **맑고 얇은 유리 구간 자체가 "여기선 숨죽이고 있었다"는 기록**이다. 실루엣과 투명도만으로 조용한 경기와 격렬한 경기가 구분되어야 한다.

## 3. 감정별 문법

세 감정은 색만 다른 세 버튼이 아니라 **서로 다른 물리 현상**이어야 한다. 부피의 방향·무늬·시간 프로파일이 모두 다르다. 축구 관전의 세 반사와 1:1 대응한다.

| | 좋아! (연타) | 안돼! (단발 탭) | 제발! (홀드) |
|---|---|---|---|
| 경기 순간 | 골, 환호 | 오심, 실축, 통탄 | 프리킥 직전, 버티는 수비 |
| 시각 동사 | 피어난다 | 움푹 물린다 | 부풀어 견딘다 |
| 부피 | **바깥으로** 둥근 융기 | **안으로** 눌린 주름 | 넓고 완만한 팽창 띠 |
| 색 | 골드→앰버, 발광 | 딥레드 코어 + 밝은 가장자리 | 블루→시안, 반투명 |
| 무늬 | 여러 개의 둥근 방울 로브 | 한 번에 하나의 깊은 접힘 | 이음새 없는 매끄러운 막 |
| 세로 발자국 | 연타 지속 구간만큼 방울방울 | 좁고 또렷한 한 켜 | 홀드 길이만큼 긴 띠 — 띠의 길이 자체가 "견딘 시간" |
| attack / 잔향 | 즉시 / 300~500ms 애프터글로우 | 즉시, 탁 접힘 / 450~700ms 안정 | 100~180ms 서서히 / 900~1400ms 길게 가라앉음 |

- **좋아!**: 탭이 누적될수록 융기가 커지고 내부 골드 발광이 밝아진다. 골 순간의 난타는 트로피에서 가장 두껍고 빛나는 금빛 마디가 된다 — 히어로 모먼트. 탭 1회에도 작은 방울 하나는 반드시 남는다(입력 강도의 위계).
- **안돼!**: 유일하게 반지름을 **줄이는** 감정. 넓은 어두운 면 + 밝은 접힘선의 명암 대비로 표현. 가는 선·가시 금지. 트로피에 남은 움푹한 상처로 읽힌다.
- **제발!**: 유일하게 **세로 길이**가 주 채널인 감정. 홀드 중에는 주조선의 용융 링이 파랗게 두꺼워지고, 그 시간에 쌓인 층 전체가 하나의 긴 팽창 띠가 된다.

### 색 규칙

- 본체 유리: 따뜻한 샴페인 골드 틴트의 반투명 유리 — 무입력 상태에서도 "금빛 트로피"로 읽히는 바탕색.
- 감정 팔레트: 골드 `#F5A524~#FFD9A0` / 레드 `#E4573D~#7A1E2B` / 블루 `#3B82F6~#A8D8FF`.
- 세 색은 **방향(θ) 기반으로 각자의 영역**을 갖는다(기존 direction-based color 계승). RGB 평균으로 보라·회색으로 뭉개는 것 금지.
- 색은 표면 스티커가 아니라 유리 내부의 깊이를 가진 층. 강한 순간일수록 더 깊은 곳까지 물든다.

## 4. 즉시 반응 — 인과의 명료함

모든 입력은 두 박자로 응답한다: **즉각 + 잔향.** 잔향이 없으면 디지털로, 있으면 마법으로 읽힌다.

```text
button input
  ├─ liveImpulse  : 같은 프레임에 주조선이 출렁임 (shader uniform만, 지오메트리 재생성 금지)
  │                 용융 링 두께 · 색 번짐 · 하이라이트 · 젖은 유리 광택
  ├─ 잔향          : 감정별 decay 프로파일(§3)로 가라앉으며
  └─ beat snapshot : 0.7s 틱에 기록 → 응고 평활화를 거쳐 영구 층으로
```

- 사용자가 1회 입력만으로 이해해야 하는 인과 사슬: **누른다 → 주조선이 즉시 출렁인다 → 그대로 굳는다 → 내 자국이 트로피에 영원히 남았다.**
- 카메라: 주조선을 화면 상단 1/3에 고정하고 아주 느리게 상승. 성장을 쫓아 프레이밍이 흔들리는 것 금지. 종료 휘슬에서만 풀백으로 트로피 전체 공개.

## 5. 재질 · 조명

- 본체: 색이 안쪽에 머무는 유리·레진 (v2 계승). `roughness 0.08~0.22, clearcoat 0.55~0.85, transmission 0.18~0.55, ior 1.35~1.48` (튜닝 범위).
- 주조선: 본체 재질 + emissive 얇은 용융 링. 평시엔 은은한 골드, 입력 시 해당 감정 색으로 달아오른다.
- 받침대: 유리가 아닌 어두운 무광 금속/석재 — 반투명 본체를 받치는 시각적 앵커.
- 조명: 밝은 회색 배경, 부드러운 키 라이트 + 차가운 블루 필 + 따뜻한 골드 림 라이트, 바닥 접지 그림자와 은은한 반사. 블룸은 주조선과 좋아! 발광에만 절제해서.
- 색·수치 토큰은 셰이더 uniform까지 배선을 확인할 것 — GLSL 하드코딩은 팔레트 수정을 무력화한다.

## 6. 결정론

- `Math.random()` 금지. 동일 `seed + beats` → 동일 실루엣·색층·하이라이트·해시 (기존 헤드리스 검증 재사용).
- 트로피 아키타입 `P(y)` 는 전원 공통. seed는 내부 색 흐름 방향, 미세 비대칭 위상만 결정한다 — "같은 경기, 다른 형상"의 차이는 seed가 아니라 **beats(감정 기록)** 에서 나온다. 라이벌 비교 뷰에서 같은 트로피 뼈대에 다른 상처·다른 금빛이 남은 것이 즉시 대비된다.
- liveImpulse의 애니메이션 시간값은 해시에 포함하지 않는다. 해시는 응고된 층 데이터에서만.

## 7. 합격 기준

구현 단계마다 스크린샷으로 검증한다. 하나라도 실패하면 다음 단계로 가지 않는다.

1. 무입력 3분 결과가 그 자체로 전시 가능한 매끈한 유리 트로피다.
2. 3버튼 스팸 60초 결과가 클램프 범위 안에 있고, 여전히 1초 안에 트로피로 읽힌다.
3. 썸네일(높이 200px)에서 실루엣·투명도만으로 조용한 경기와 격렬한 경기가 구분된다.
4. 처음 보는 사람이 설명 없이 "아래가 킥오프, 위가 종료"를 읽는다.
5. 그레이스케일로 봐도 세 감정의 흔적이 부피·무늬로 구분된다.
6. 입력 1회 → 주조선 반응까지 한 프레임. 다음 틱을 기다리게 하지 않는다.
7. 회전시키면 앞·뒤의 색 영역이 다르게 보인다 (방향 기반 색의 존재 증명).
8. 층 경계가 지오메트리 단차로 보이지 않는다.
9. 받침대·스템·컵이 분리된 파츠가 아니라 하나의 주조물로 읽힌다.

## 8. 데모 연출 (15초 머니샷)

빈 받침대 위로 금빛 유리가 차오르기 시작한다 → 파란 홀드가 스템을 길게 감싸며 부푼다 → 레드가 컵 아랫배를 한 번 탁 문다 → 골드 연타가 주조선을 달구며 컵 상단이 방울방울 피어난다 → 림까지 차오르고 휘슬, 카메라 풀백 — 90분의 감정이 새겨진 트로피 완성 → 라이벌 팬의 트로피가 옆에 선다: 같은 뼈대, 정반대의 상처 → 받침대에 해시 각인, Mint.

## 10. v3.1 — 파티클 주조 부록

v3 유리 메시 구현이 "그럭저럭"에 머문 원인: 정적인 솔리드는 열정의 에너지를 전달하지 못했다. 재료를 입자로 바꾼다. 아래 항목 외의 모든 규칙(아키타입 P(u), 시간축, 감정 문법의 방향·색·잔향, 결정론, 합격 기준)은 v3 그대로 유효하다.

### 재료 = 입자

- **고스트 아키타입**: 입력이 없어도 트로피가 보여야 하므로, 저폴리 삼각 패싯 메시(어두운 필 + 와이어프레임 글로우)가 아키타입 실루엣을 상시 담당한다. 감정 입자는 이 뼈대 위에 얹힌다. 고요한 틱은 표면의 미광 더스트만 남긴다.
- 입력별 입자 문법 — 세 감정은 입자의 **탄생 방식**이 다르다:
  - **좋아!** 한 점에서 터지는 스플랫 → 크고 밝은 금빛 방울 클러스터로 정착 (히어로 입자 — 가장 큰 사이즈)
  - **안돼!** 바깥의 흩어진 점들이 안으로 스냅 → 붉은 세로 시임
  - **제발!** 홀드하는 동안 얇은 글로우 라인이 그려지듯 트로피를 감아 올라간다 — 틱마다 시작각이 전진해 긴 홀드는 나선 실이 된다. 나선의 길이 = 견딘 시간
- **그래디언트 쌍**: 이벤트마다 감정 램프에서 오리진-인접 색 쌍을 결정론적으로 뽑아 입자별로 섞는다. 단색 감정 금지 — 살아있는 그라데이션.
- **휘휘 젓기**: 완성 후 포인터를 저으면 주변 입자가 소용돌이로 흩어졌다 홈 위치로 복귀한다. 흩어져도 다시 모인다 = 기록은 지워지지 않는다는 메타포.
- **완성 쇼크웨이브**: 종료 휘슬에 전체 입자가 한 번 퍼졌다 모이며 주조 완료를 선언한다.

### 무대 전환 — 스타디움 나이트

가산 블렌딩 발광 입자는 밝은 배경에서 죽는다. 배경을 어두운 네이비(#0A0D18)로, 블룸을 상시로 전환한다. v3 §5의 "밝은 회색 배경" 조항은 폐기.

### 성능 규칙

- 정착·산란·복귀·반짝임은 전부 버텍스 셰이더. CPU는 입력 순간에 새 입자 속성만 쓴다(부분 버퍼 업로드).
- 입자 풀 상한 150,000. 초과 시 새 입자 생성을 멈추되 실루엣은 이미 완성돼 있다.
- 결정론 대상은 홈 위치·색 쌍·크기. 스폰 위치·생성시각은 애니메이션 전용으로 해시에서 제외.

## 9. 구현 순서

1. 감정 미연결 상태의 트로피 본체(`P(y)` + 받침대) + 유리 재질 — **입력 없이 아름다울 때까지 여기서 멈춘다.**
2. 주조 애니메이션(아래→위 차오름) + 주조선 용융 링.
3. 응고 평활화 파이프라인 (세로 번짐 + 둘레 피처 폭 제한 + 클램프).
4. 세 감정의 층 문법 (부피 방향·무늬·색층).
5. liveImpulse + 잔향 프로파일.
6. beats 리샘플링 → 결정론·해시 검증.
7. 카메라 연출 · 라이벌 비교 뷰 · 받침대 각인 · 민팅 화면 연결.
